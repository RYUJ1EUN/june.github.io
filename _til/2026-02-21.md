#### 1. GitHub Actions와 DevOps

GitHub Actions: 코드 저장소 기능 + 소프트웨어 빌드/테스트/배포 자동화 ⇒ DevOps 핵심 도구

- CI: 코드 변경 시 자동으로 빌드 및 테스트 수행
- CD: 검증된 코드를 서버나 클라우드에 자동으로 반영


#### 2. DevOps 툴체인 내 위치

| 분류 | 주요 도구 | GitHub Actions의 역할 |
| --- | --- | --- |
| CI/CD | **GitHub Actions**, Jenkins, GitLab CI | 자동화 워크플로우 실행 엔진 |
| 인프라 관리 | Terraform, Ansible | 배포 스크립트 실행 및 인프라 변경 적용 |
| 협업/운영 | Slack, Jira, Prometheus | 작업 완료 알림 및 상태 모니터링 연동 |

<br>

#### 3. GitHub Actions의 기본 작동 모델

이벤트 발생(Trigger) → 실행 환경 설정(Runner) → 단계별 작업(Steps) 수행


##### 간단한 워크플로우 예시

`.github/workflows/main.yml`가 존재하는 상황  
_(YAML: JSON의 상위 집합 Markup 언어의 일종)_

```yaml
name: Hello DevOps Robot

# 1. Trigger: 언제 실행할 것인가?
on:
  push:
    branches: [ "main" ]

# 2-3. Jobs: 어떤 환경에서 무엇을 할 것인가?
jobs:
  build:
    runs-on: ubuntu-latest # Runner: 가상 우분투 환경

    steps:
      - name: Checkout code
        uses: actions/checkout@v4 # 작업: 소스 코드 로드

      - name: Run Greeting
        run: echo "Hello, DevOps World!" # 작업: 명령어 실행

```



##### 자동화 프로세스 흐름

1. 코드 푸시: 개발자가 `main` 브랜치에 코드를 올림
2. 이벤트 감지: GitHub이 푸시를 감지하고 워크플로우 가동
3. 러너 할당: GitHub이 가상 머신을 배정
4. 단계별 실행: YAML에 정의된 순서대로 테스트, 빌드 등 수행
5. 결과 보고: 성공/실패 여부를 GitHub Actions 탭에 표시
